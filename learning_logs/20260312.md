App.tsx 초기화

```tsx
import { Container, AppBar, Toolbar, Typography, CssBaseline } from "@mui/material"

function App() {
  return (
    <Container maxWidth='xl'>
    <CssBaseline />
      <AppBar position="static">
        <Toolbar>
          <Typography variant="h6">
            Car Shop
          </Typography>
        </Toolbar>
      </AppBar>
    </Container>
  )
}

export default App

```
- 저희가 다양한 코드에서 확인했듯이 MUI Container 컴포넌트 내에 AppBar를 하위 컴포넌트로 위치시킴.
  (props drilling 개념 때문에 매우 중요함.)
  position props을 이용하여 AppBar의 위치를 static으로 잡음.
    - static : 스크롤할 때 앱 바가 상단에 고정되지 않고 사라지게 됨
    - fixed : 스크롤 해서 넘어가더라도 앱 바가 상단에 남아있음.

    - maxWidth : 영어 읽으면 답 나오고, 이미 shoppinglist / todolist에서 사용했음.
      (camelCase인 것만 신경쓰겠음.)

    - CssBaseline 컴포넌트를 이용하여 여러 브라우저에서 리액트 앱의 형태가 균일하게 유지되도록 설정함.
      일반적으로 최상단에 위치시켜 하위 컴포넌트들이 동일한 영향을 받도록 사용함.

    - 여기서 제가 AppBar의 position을 속성이라고 하지 않고 프롭이자고 이야기한 부분에 주목
      html에서의 속성은 attribute 였음. 하지만 props drilling에서의 props의 full word는
      properties임.
